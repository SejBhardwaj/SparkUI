# API Consistency Guidelines

## Overview
Guidelines for maintaining consistent component APIs across the library.

## Naming Conventions
- Props should use camelCase
- Event handlers should start with 'on'
- Boolean props should use 'is' or 'has' prefix

## Common Props
- className: for custom styling
- children: for content composition
- disabled: for disabling interactions
- variant: for visual variations
- size: for size variations

## Best Practices
Consistent prop naming improves developer experience.
